# Medical KAG System - 요구사항 정의서 (PRD)

> **Version**: 1.29.0

## 프로젝트 개요

**한 줄 요약**: SNOMED-CT 의학 온톨로지와 KAG(Knowledge Augmented Generation) 프레임워크를 결합하여, 의학 논문 PDF를 기반으로 정확한 의학 논문 작성을 지원하는 지식 증강 시스템

## 핵심 타겟 (페르소나)

### 주 사용자: 의학 연구자/논문 작성자
- 의학 논문을 읽고 새로운 논문을 작성해야 하는 연구자
- 다수의 참고 논문에서 관련 정보를 빠르게 찾아야 함
- 정확한 의학 용어 사용과 올바른 인용이 필수
- 복잡한 의학적 개념 간의 관계를 이해해야 함

### 사용 규모
- 개인/소규모 연구팀용 (로컬 실행)
- 100~1,000개 의학 논문 PDF 처리

## 문제 정의

```

│  3. 다단계 추론 불가                                              │
│     └─ "A 약물이 B 질환에 효과적인가?" 같은 복합 질문 처리 어려움   │
│                                                                 │
│  4. 동의어/약어 처리 미흡                                         │
│     └─ "MI", "Myocardial Infarction", "심근경색" 연결 안됨        │
│                                                                 │
│  5. 인용 vs 원본 구분 불가 (NEW)                                  │
│     └─ 논문 A가 인용한 논문 B의 내용을 A의 결과로 오인              │
│                                                                 │
│  6. 정보 밀도 차이 무시 (NEW)                                     │
│     └─ Methods 섹션이 Results만큼 검색되어 노이즈 발생             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 주요 기능 (MVP Scope)

### 필수 기능 1: 의학 논문 PDF 지식 추출 및 인덱싱

```
[PDF 입력]
     ↓
[텍스트 추출 + 섹션 분류]
     ↓
┌────────────────────────────────────────┐
│         섹션별 분류                     │
├────────────────────────────────────────┤
│  Tier 1 (핵심)     │  Tier 2 (상세)    │
│  ─────────────────│──────────────────  │
│  • Abstract       │  • Introduction   │
│  • Results        │  • Methods        │
│  • Conclusion     │  • Discussion     │
│  • Tables/Figures │  • References     │
└────────────────────────────────────────┘
     ↓
[의학 엔티티 추출 + 인용 감지]
     ↓
┌────────────────────────────────────────┐
│         출처 유형 분류                  │
├────────────────────────────────────────┤
│  • original: 해당 논문의 원본 결과      │
│  • citation: 다른 논문 인용 내용        │
│  • background: 일반적 배경 지식         │
└────────────────────────────────────────┘
     ↓
[SNOMED-CT 매핑] → [지식그래프 저장]
     ↓
[벡터 임베딩] → [벡터DB 저장]
```

**세부 기능:**
- PDF에서 텍스트, 표, 그림 설명 추출
- **논문 섹션 자동 분류** (IMRAD 구조 인식)
- **인용 패턴 감지** `(Author, 2023)`, `[1-3]` 등
- 의학 용어(질병, 약물, 증상, 치료법 등) 자동 추출 (NER)
- 추출된 용어를 SNOMED-CT 개념에 매핑
- 개념 간 관계(원인-결과, 치료-질병 등)를 지식그래프로 구축
- 동시에 텍스트 청크를 벡터로 저장 (하이브리드 검색용)

### 필수 기능 2: 하이브리드 검색 (벡터 + 지식그래프)

```
[사용자 쿼리]
      ↓
[쿼리 분석 + 의도 파악]
      ↓
┌─────────────────┐    ┌─────────────────┐
│   벡터 검색     │    │  지식그래프 검색  │
│  (의미 유사도)   │    │  (구조적 관계)   │
└────────┬────────┘    └────────┬────────┘
         │                      │
         └──────────┬───────────┘
                    ↓
         [Tier 1 (핵심) 우선 검색]
                    ↓
         [부족시 Tier 2 확장 검색]
                    ↓
            [결과 융합 & 랭킹]
                    ↓
┌─────────────────────────────────────────┐
│            랭킹 요소                     │
├─────────────────────────────────────────┤
│  • 관련도 점수                          │
│  • 출처 유형 (original > citation)      │
│  • 근거 수준 (RCT > Cohort > Case)      │
│  • 시간 가중치 (최신 논문 우선)          │
└─────────────────────────────────────────┘
                    ↓
            [컨텍스트 생성]
```

**세부 기능:**
- 쿼리에서 의학 엔티티 자동 추출
- SNOMED-CT 기반 쿼리 확장 (동의어, 상위/하위 개념)
- 벡터 유사도 검색 + 지식그래프 경로 검색 병행
- **Tier 1 우선 검색**, 필요시 Tier 2 확장
- **원본 결과 우선** 랭킹 (인용 내용은 순위 하향)
- **근거 수준 반영** (RCT > 관찰연구 > 사례보고)
- 두 결과를 융합하여 최종 랭킹

### 필수 기능 3: 논문 작성 지원 (논리적 추론)

```
[연구 질문 입력]
      ↓
[다단계 추론 계획 수립]
      ↓
[단계별 검색 및 추론 실행]
      ↓
┌─────────────────────────────────────────┐
│         추론 결과 구성                   │
├─────────────────────────────────────────┤
│  • 근거 기반 답변                       │
│  • 추론 경로 설명                       │
│  • 상충 결과 표시 (있을 경우)            │
│  • 근거 수준별 정리                     │
└─────────────────────────────────────────┘
      ↓
[인용 정보 자동 첨부]
      ↓
┌─────────────────────────────────────────┐
│         인용 품질 보장                   │
├─────────────────────────────────────────┤
│  • 원본 논문만 인용 (2차 인용 방지)      │

│  • 페이지 번호 포함                     │

```

**세부 기능:**
- 복잡한 연구 질문을 하위 질문으로 분해
- 각 하위 질문에 대해 검색 및 추론 수행
- **상충되는 결과 자동 감지 및 표시**
- 논문에서 근거를 찾아 답변 생성
- **원본 출처만 인용** (2차 인용 오류 방지)
- 사용된 모든 출처에 대해 정확한 인용 정보 제공

### v3.1 Reasoning & Intelligence 고도화 (계획)

#### Phase 1: 코드 품질 개선 (1주)
- [ ] **Entity Normalizer 퍼지 매칭**: Edit Distance 기반 유사도 매칭 (rapidfuzz)
  - 변형 처리: "T-LIF" ↔ "TLIF" ↔ "Trans-LIF"
  - 토큰 기반 매칭: 단어 순서 무관
  - 약어 확장: "TL" → "TLIF"
- [ ] **Direction Determination 개선**: 결과 방향 추론 강화
  - 값 비교: intervention vs control 수치 비교
  - 결과 유형별 방향 해석 (pain은 감소가 개선, function은 증가가 개선)
  - 텍스트 키워드 기반 방향 추출
- [ ] **Hybrid Ranker 동적 가중치**: 쿼리 유형별 가중치 조정
  - comparison 쿼리: Graph 70% + Vector 30%
  - mechanism 쿼리: Graph 30% + Vector 70%
  - evidence 쿼리: Graph 80% + Vector 20%

#### Phase 2: 핵심 기능 추가 (2주)
- [ ] **Conflict Detection & Resolution**: 상충 연구 자동 탐지
  - 동일 Intervention-Outcome 쌍에 대한 상반된 결과 탐지
  - 상충 이유 추론 (sample size, evidence level, follow-up 차이)
  - 규칙 기반 권고안 생성
- [ ] **Cypher 기반 추론 규칙**: Neo4j에서 추론 관계 생성
  - 전이적 치료 관계: A→treats→B→causes→C ⇒ A→indirectly_affects→C
  - 비교 대상 추론: 같은 질환 치료 수술법들 자동 연결
  - 계층적 근거 상속: 하위 수술법 근거를 상위 카테고리에 적용
- [ ] **Evidence Synthesis**: 메타분석 수준 근거 종합
  - 가중 평균 효과 크기 계산
  - 이질성 검정 (I² statistic)
  - GRADE 기준 근거 수준 평가

## LLM 기반 처리 접근법 (v2.0)

### 개요

규칙 기반 처리의 한계를 극복하기 위해 **Gemini 2.5 Flash LLM**을 활용한 처리 파이프라인을 도입합니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    LLM 기반 처리 파이프라인                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [PDF 입력]                                                     │
│       ↓                                                         │
```

## PubMed 통합 (v2.0 구현 완료)

### 개요

실시간 PubMed API 연동을 통해 최신 의학 논문을 자동으로 검색, 다운로드, 처리할 수 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    PubMed 통합 워크플로우                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [검색 쿼리 입력]                                                │
│       ↓                                                         │
│  [PubMed API 검색]                                              │
│       ├─ 키워드 기반 검색                                        │
│       ├─ 필터링 (날짜, 논문 유형)                                │
│       └─ Async 배치 처리 (성능 최적화)                           │
│       ↓                                                         │
│  [메타데이터 수집]                                               │
│       ├─ 제목, 저자, 초록                                        │
│       ├─ 발행일, 저널명                                          │
│       ├─ PMID, DOI                                              │
│       └─ MeSH 용어 (의학 주제)                                   │
│       ↓                                                         │
│  [초록 기반 사전 필터링]                                          │
│       ├─ 관련성 평가                                             │
│       └─ 중복 제거                                               │
│       ↓                                                         │
│  [지식 베이스 통합]                                              │
│       ├─ 메타데이터 저장                                         │
│       ├─ 초록 벡터 임베딩                                        │
│       └─ 논문 관계 그래프 업데이트                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 주요 기능

**1. PubMed 검색**
- 키워드, 저자, 저널명 등 다양한 검색 옵션
- 날짜 범위, 논문 유형 필터링
- 결과 제한 및 정렬 옵션

**2. 메타데이터 자동 수집**
- NCBI E-utilities API 활용
- 논문 정보 자동 파싱
- MeSH 용어 자동 추출

**3. 비동기 배치 처리**
- 대량 논문 검색 시 성능 최적화
- Rate limiting 준수 (NCBI 정책)
- 실패 시 자동 재시도

**4. 지식 베이스 통합**
- 검색 결과를 기존 지식 베이스에 자동 통합
- 중복 논문 자동 감지
- 논문 간 인용 관계 업데이트

**5. Fulltext 자동 조회 (v1.13 신규)**
- 3단계 Fallback 체인으로 전문 자동 수집:
  1. **PMC Open Access**: BioC API로 구조화된 전문 조회
  2. **DOI/Unpaywall**: Crossref + Unpaywall API로 OA 상태 확인 및 전문 조회
  3. **Abstract Only**: 전문 미확보 시 초록만 저장
- Open Access 상태 분류: gold, green, hybrid, bronze, closed
- DOI 메타데이터 자동 저장

### 성능 최적화

| 최적화 기법 | 설명 |
|------------|------|
| Async I/O | aiohttp를 통한 비동기 API 호출 |
| 배치 처리 | 여러 논문을 한 번에 처리 |
| 캐싱 | 중복 쿼리 결과 재사용 |
| Rate Limiting | API 제한 준수 (3 requests/sec) |

### 사용 예시

```python
# PubMed에서 논문 검색
results = await pubmed_client.search(
    query="diabetes metformin kidney",
    max_results=50,
    date_range=("2020/01/01", "2024/12/31")
)

# 메타데이터 자동 수집
papers = await pubmed_client.fetch_details(results)

# 지식 베이스에 추가
for paper in papers:
    await knowledge_base.add_paper(paper)
```

```

      ├─ 연구 설계 분류 (RCT, Cohort 등)
      ├─ PICO 요소 추출
      ├─ SNOMED-CT 매핑 (당뇨병 → 73211009)

   └─ "논문 추가 완료:

       - 의학 개념: 156개 (SNOMED 매핑 89개)
       - 원본 결과: 42개 / 인용 내용: 23개

```

Streamlit의 비동기 이벤트 루프 제약으로 동기 Neo4j 드라이버 사용:

```python
# web/utils/graph_utils.py
from neo4j import GraphDatabase  # 동기 드라이버

class SyncNeo4jClient:
    def run_query(self, cypher: str, params: dict = None) -> list[dict]:
        with self._driver.session(database=self.database) as session:
            result = session.run(cypher, params or {})
            return [dict(record) for record in result]
```
